# 🖥️ GitHub AutoSync 图形界面版

## ✨ 功能特点

- 🎨 **苹果质感界面** - 蓝白配色，现代化设计
- 🖱️ **可视化操作** - 无需命令行，点击即可使用
- 🔄 **实时状态** - 同步状态、日志实时显示
- 🖱️ **Cursor 联动** - 自动检测 Cursor 运行状态
- 📊 **数据统计** - 同步次数、文件监听等统计信息

## 🚀 快速开始

### 方式一：一键启动（推荐）

双击运行：**`启动应用.bat`**

### 方式二：命令行启动

```powershell
cd app
npm install
npm start
```

## 📋 使用步骤

### 1. 首次配置

1. 打开应用后，点击左侧 **"配置"** 菜单
2. 点击 **"浏览"** 选择你的项目目录
3. 填写 **GitHub Token**（点击"获取 Token"按钮可跳转到 GitHub）
4. 填写 **GitHub 仓库地址**（如果已配置 Git，会自动检测）
5. 设置 **防抖时间**（默认10秒）
6. 点击 **"保存配置"**

### 2. 启动同步

1. 返回 **"首页"**
2. 点击 **"启动自动同步"** 按钮
3. 状态显示为 **"运行中"** 即表示已启动

### 3. 查看状态

- **状态页面**：查看同步状态、项目目录、远程仓库等信息
- **日志页面**：查看实时同步日志

## 🎯 与 Cursor 联动

应用会自动检测 Cursor 是否运行：
- ✅ Cursor 运行中：显示绿色指示器
- ❌ Cursor 未运行：显示灰色指示器

在 Cursor 中编辑代码时，应用会自动同步到 GitHub。

## 🐛 常见问题

### Q: 启动失败？

**A:** 
1. 确保已安装 Node.js
2. 运行 `npm install` 安装依赖
3. 检查 PowerShell 执行策略

### Q: 无法检测到 Cursor？

**A:** 
- 确保 Cursor 正在运行
- Windows 系统会自动检测，其他系统可能需要手动配置

### Q: 同步服务无法启动？

**A:** 
1. 检查配置是否完整
2. 检查 Git 是否安装
3. 检查项目目录是否正确
4. 查看日志页面的错误信息
