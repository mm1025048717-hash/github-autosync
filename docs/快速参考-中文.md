# ⚡ 快速参考 - GitHub AutoSync（中文版）

## 🎯 一句话说明

这是一个**自动同步工具**，让你一边在 Cursor 中聊天写代码，一边自动将代码同步到 GitHub。

## 🚀 3步快速开始

```powershell
# 1. 安装
.\install.ps1

# 2. 设置 Token
$env:GITHUB_TOKEN = "ghp_你的token"

# 3. 启动
.\start.ps1
```

## 📁 核心文件（记住这5个）

| 文件名 | 中文名 | 做什么 | 什么时候用 |
|--------|--------|--------|-----------|
| `auto-sync.ps1` | **自动同步主程序** | 监听文件，自动提交推送 | ⭐核心，最重要 |
| `start.ps1` | **启动脚本** | 启动服务 | 每天开始工作时 |
| `stop.ps1` | **停止脚本** | 停止服务 | 结束工作时 |
| `deploy-with-token.ps1` | **部署脚本** | 首次部署到 GitHub | 第一次使用时 |
| `install.ps1` | **安装脚本** | 自动安装配置 | 第一次使用时 |

## 🎮 常用命令

```powershell
# 启动自动同步
.\start.ps1

# 停止自动同步
.\stop.ps1
# 或按 Ctrl+C

# 首次部署
.\deploy-with-token.ps1 -RepoUrl "https://github.com/用户名/仓库名.git" -Token "ghp_token"
```

## ❓ 常见问题速查

### 文件变化后没有自动提交？
- 检查防抖时间（默认10秒）
- 检查文件是否在排除列表
- 检查服务是否运行

### 如何获取 Token？
1. 访问 https://github.com/settings/tokens
2. 生成新 Token，勾选 `repo` 权限

### 如何停止服务？
- 前台：按 `Ctrl+C`
- 后台：运行 `.\stop.ps1`

## ⚙️ 配置文件

创建 `config.json`（复制 `config.json.example`）：

```json
{
  "github": {
    "repository": "https://github.com/用户名/仓库名.git",
    "token": "ghp_你的token"
  },
  "sync": {
    "debounceSeconds": 10
  }
}
```

## 🎯 使用场景

1. **与 AI 聊天写代码** - 在 Cursor 中聊天，代码自动同步
2. **多设备协作** - 一台设备编辑，其他设备拉取
3. **自动备份** - 每次保存自动提交

- ⭐ **核心文件**：`auto-sync.ps1` 是最重要的
