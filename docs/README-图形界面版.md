# 🖥️ GitHub AutoSync - 图形界面版使用指南

## 🎉 新功能：图形界面！

现在 GitHub AutoSync 有了**漂亮的图形界面**，无需命令行，点击即可使用！

## ✨ 界面特点

- 🎨 **苹果质感设计** - 蓝白配色，现代化界面
- 🖱️ **可视化操作** - 所有功能都可以点击完成
- 📊 **实时状态显示** - 同步状态、日志实时更新
- 🖱️ **Cursor 联动检测** - 自动检测 Cursor 是否运行
- 📈 **数据统计** - 同步次数、文件监听等统计

## 🚀 快速开始

### 方式一：一键启动（最简单）

1. 进入 `app` 目录
2. 双击运行 **`启动应用.bat`**
3. 应用会自动安装依赖并启动

### 方式二：命令行启动

```powershell
cd app
npm install
npm start
```

## 📖 使用流程

### 第一步：配置设置

1. 打开应用后，点击左侧 **"配置"** 菜单
2. 点击 **"浏览"** 按钮，选择你的项目目录
3. 填写 **GitHub Token**：
   - 点击 **"获取 Token"** 按钮跳转到 GitHub
   - 生成 Token 后复制粘贴到输入框
4. **GitHub 仓库地址**会自动检测（如果已配置 Git）
5. 设置 **防抖时间**（默认10秒）
6. 点击 **"保存配置"**

### 第二步：启动同步

1. 返回 **"首页"**
2. 点击 **"▶️ 启动自动同步"** 按钮
3. 状态显示为 **"运行中"** 即表示成功启动

### 第三步：开始编码

现在你可以：
- ✅ 在 Cursor 中编辑代码
- ✅ 保存文件后自动同步到 GitHub
- ✅ 在应用的 **"日志"** 页面查看同步记录

## 🎯 界面说明

### 🏠 首页
- **同步状态指示器** - 显示当前同步状态
- **启动/停止按钮** - 控制同步服务
- **快速统计** - 同步次数、监听文件数、最后同步时间

### ⚙️ 配置页
- **项目目录** - 选择要同步的项目
- **GitHub Token** - 配置认证 Token
- **仓库地址** - GitHub 仓库 URL
- **防抖时间** - 文件变化后等待时间

### 📊 状态页
- **同步状态** - 当前是否运行
- **项目信息** - 目录、仓库、防抖时间
- **Cursor 联动** - Cursor 运行状态检测

### 📝 日志页
- **实时日志** - 同步过程的详细日志
- **清空日志** - 一键清空所有日志

## 🖱️ 与 Cursor 联动

应用会自动检测 Cursor 是否运行：
- ✅ **Cursor 运行中** - 显示绿色指示器
- ❌ **Cursor 未运行** - 显示灰色指示器

在 Cursor 中编辑代码时，应用会自动将更改同步到 GitHub。

## 💡 使用技巧

1. **首次使用**：先配置设置，再启动同步
2. **查看日志**：遇到问题时查看日志页面的错误信息
3. **测试连接**：配置完成后点击"测试连接"验证配置
4. **自动检测**：选择项目目录后会自动检测 Git 仓库

## 🆚 命令行版 vs 图形界面版

| 功能 | 命令行版 | 图形界面版 |
|------|---------|-----------|
| 启动方式 | 运行脚本 | 双击启动 |
| 配置方式 | 编辑文件 | 可视化配置 |
| 状态查看 | 查看终端 | 界面显示 |
| 日志查看 | 终端输出 | 日志页面 |
| 使用难度 | 中等 | 简单 |

**推荐：新手使用图形界面版，高级用户可以使用命令行版**
